# AI Era Skills — Overview

How AI is reshaping every engineering path. For each track, skills are classified into four categories.

---

## How to read these pages

!!! success "New AI-era skills"
    Things you need that didn't exist or weren't mainstream 3 years ago. These are additive — new skills on top of existing ones.

!!! info "Amplified importance"
    Skills that matter *more* because of AI. When AI handles boilerplate, judgment and architecture become more valuable, not less.

!!! example "Still essential"
    The fundamentals that aren't going anywhere. These remain the foundation regardless of AI's progress.

!!! warning "Increasingly automated"
    Where AI handles the routine work. These skills don't disappear, but you stop getting much credit for them — they become table stakes.

---

## Big-picture themes

**The skills being automated** are rarely disappearing entirely — they're becoming things AI handles in seconds, so you stop getting credit for them. Value moves up the stack to judgment, architecture, and the ability to evaluate AI output critically.

**The skills marked amplified** are getting harder to fake. When AI writes the boilerplate, the things that separate engineers are taste, system-level thinking, security awareness, and the ability to spot when generated code is subtly wrong.

**The cross-cutting skill** that appears in every path is the ability to design and evaluate systems that include AI as a component — not just call an API and hope for the best.
