# AI Era — Frontend Engineer

> **Bottom line:** The bar for shipping UI has risen. AI can generate layouts fast — your edge is taste, judgment, and building features *around* AI (chat UIs, streaming, agentic interfaces).

---

!!! success "New AI-era skills"
    - AI UI tools (v0, Bolt, Cursor)
    - Streaming UI patterns
    - LLM API integration
    - Designing chat interfaces
    - AI-powered search & autocomplete
    - React Server Components for AI

!!! info "Amplified importance"
    - System design thinking
    - Accessibility & WCAG
    - Performance optimization
    - TypeScript
    - Testing (e2e)
    - UX judgment

!!! example "Still essential"
    - React / Vue
    - CSS & responsive design
    - REST / GraphQL
    - Git

!!! warning "Increasingly automated"
    - Boilerplate component code
    - Basic CRUD forms
    - Initial scaffolding
    - Copy-paste CSS patterns

## What this means in practice

The frontend role is bifurcating. Engineers who use AI tools to ship faster *and* retain sharp judgment about what good UI looks like will thrive. Engineers who either resist AI tools entirely or blindly ship AI-generated code without critique will struggle.

The most valuable new skill is building AI-native UI patterns — streaming text responses, chat interfaces, multi-step agentic flows, and real-time collaborative features. These don't have established patterns yet, which means engineers who figure them out now are writing the playbook.
