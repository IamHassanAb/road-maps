## Learning path

1. **Swift or Kotlin fundamentals** *(2‚Äì3 months ¬∑ Free)*
   Apple's "Develop in Swift" or Google's Android Basics in Kotlin ‚Äî official, free, excellent.

2. **SwiftUI or Jetpack Compose** *(1‚Äì2 months ¬∑ Free)*
   Hacking with Swift (Paul Hudson) for iOS. Google's Compose pathway for Android.

3. **Networking & data persistence** *(3‚Äì4 weeks ¬∑ Free)*
   URLSession + Codable (iOS) or Retrofit + Room (Android). Build an app that fetches and caches data.

4. **App architecture** *(3‚Äì4 weeks ¬∑ Mixed)*
   MVVM, Clean Architecture, or TCA (iOS). Architecture determines everything at scale.

5. **Cross-platform with React Native or Flutter** *(1‚Äì2 months ¬∑ Free)*
   Expo docs + build a production-quality cross-platform app.

6. **App Store deployment & CI/CD** *(1‚Äì2 weeks ¬∑ Free)*
   Fastlane docs + set up an automated pipeline: test ‚Üí build ‚Üí deploy.

7. **On-device ML** *(2‚Äì3 weeks ¬∑ Free)*
   Apple's CoreML & Create ML tutorials. Build a real app that classifies images or runs speech-to-text.

## Interview topics

=== "Swift / Kotlin"
    - ARC & memory management
    - Concurrency (async/await, coroutines)
    - Value vs reference types
    - Protocol-oriented (Swift) / interfaces (Kotlin)
    - Error handling patterns

=== "UI & layout"
    - Auto Layout constraints (iOS)
    - ConstraintLayout (Android)
    - SwiftUI state management
    - Animations & transitions
    - Accessibility on mobile

=== "Architecture"
    - MVVM vs MVC vs TCA
    - Dependency injection
    - Repository pattern
    - Offline-first design
    - App modularization

=== "Platform specifics"
    - App lifecycle (foreground/background)
    - Push notification flow
    - App Store review process
    - Keychain & secure storage
    - Deep linking architecture
