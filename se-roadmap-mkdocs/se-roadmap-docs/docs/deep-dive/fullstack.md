# Deep Dive — Full Stack Engineer

---

## Career levels

=== "Junior (0–2 yrs)"
    **Focus:** Build complete small features (UI + API + DB), work on well-defined tickets.

    `React basics` `Node.js or Python backend` `REST API design` `SQL fundamentals` `Git` `HTML & CSS` `Basic deployment`

=== "Mid-level (2–5 yrs)"
    **Focus:** Own entire product features from DB schema to UI, improve team velocity.

    `TypeScript (front & back)` `PostgreSQL + ORMs` `Docker basics` `Auth (JWT/OAuth)` `Testing (unit + e2e)` `State management` `API design patterns`

=== "Senior (5–10 yrs)"
    **Focus:** Architect full systems, make tech stack decisions, define patterns used across the product.

    `System architecture` `Database optimization` `Next.js (full-stack)` `GraphQL or tRPC` `Performance optimization` `Security design` `Cloud deployment` `Mentoring juniors`

=== "Staff / Principal (10+ yrs)"
    **Focus:** Define the technical roadmap, lead cross-functional initiatives, set engineering culture.

    `Platform architecture` `AI feature strategy` `Build vs buy decisions` `Technical leadership` `Developer experience` `Product-engineering alignment`

## Learning path

1. **HTML, CSS, JS + React** *(3–4 months · Free)* — The Odin Project is a great free structured path.
2. **Backend with Node.js or Python** *(2–3 months · Free)* — Build a REST API with a real database.
3. **TypeScript (both ends)** *(3–4 weeks · Paid)* — Total TypeScript + ts-node for backend.
4. **Database design** *(3–4 weeks · Free)* — SQL properly, then an ORM (Prisma). Understand indexes.
5. **Full-stack frameworks (Next.js)** *(1–2 months · Free)* — Build a full SaaS app with App Router, Server Actions, and Prisma.
6. **Authentication deep dive** *(1–2 weeks · Free)* — Implement OAuth2 from scratch once, then use NextAuth.
7. **AI feature integration** *(2–3 weeks · Free)* — Add LLM chat, vector search, and streaming to a project.

## Interview topics

=== "JavaScript / TypeScript"
    - Async/await & Promises
    - TypeScript generics
    - Event loop
    - ES6+ patterns
    - Error handling

=== "System design"
    - Design a SaaS product
    - URL shortener
    - Real-time features
    - Auth system design
    - File upload system

=== "React + backend"
    - Data fetching patterns
    - Server vs client components
    - API rate limiting
    - Database N+1 problem
    - Caching strategies

=== "Product thinking"
    - Feature scoping & trade-offs
    - Performance vs developer UX
    - Schema evolution
    - Scaling a monolith
    - Testing strategy
