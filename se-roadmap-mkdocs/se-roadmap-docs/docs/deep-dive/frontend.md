# Deep Dive — Frontend Engineer

---

## Career levels

=== "Junior (0–2 yrs)"

    **Focus:** Build UI components from designs, fix bugs, write clean HTML/CSS/JS.

    `HTML & CSS fundamentals` `JavaScript basics` `React (hooks, state)` `Git workflows` `Browser DevTools` `REST API consumption` `Responsive layouts`

=== "Mid-level (2–5 yrs)"

    **Focus:** Own features end-to-end, mentor juniors, improve code quality and performance.

    `TypeScript` `State management (Redux/Zustand)` `Component architecture` `Testing (Jest, RTL)` `Web performance (Core Web Vitals)` `Accessibility (WCAG AA)` `CI/CD basics` `Design systems`

=== "Senior (5–10 yrs)"

    **Focus:** Drive architecture decisions, define patterns, unblock the team, own complex features.

    `System design for frontend` `Micro-frontends` `SSR/SSG/ISR (Next.js)` `Advanced TypeScript` `Bundle optimization` `GraphQL` `Security (XSS, CSRF)` `Mentoring`

=== "Staff / Principal (10+ yrs)"

    **Focus:** Set technical direction across teams, shape engineering culture, drive org-wide decisions.

    `Frontend platform strategy` `Cross-team API contracts` `Performance budgets at scale` `Developer tooling & DX` `AI-assisted UI toolchains` `RFC & technical writing` `Org-level mentoring`

## Learning path

1. **HTML, CSS & JS fundamentals** *(2–3 months · Free)*
   MDN Web Docs, The Odin Project, or freeCodeCamp. Non-negotiable foundation.

2. **React & component thinking** *(1–2 months · Free)*
   Official React docs (react.dev) + build 3–5 real projects with hooks and state.

3. **TypeScript** *(3–4 weeks · Paid)*
   Total TypeScript by Matt Pocock — best structured course available.

4. **Next.js & full-stack patterns** *(3–4 weeks · Free)*
   Official Next.js docs + Vercel tutorials. Learn App Router, RSC, and deployment.

5. **Testing** *(2–3 weeks · Free/Paid)*
   Kent C. Dodds' Testing Trophy approach — start with integration tests.

6. **Performance & accessibility** *(2–3 weeks · Free)*
   web.dev/learn + Lighthouse auditing. Study Core Web Vitals and ARIA.

7. **System design for frontend** *(Ongoing · Mixed)*
   ByteByteGo resources + build complex apps. Read about micro-frontend architecture.

## Interview topics

=== "JavaScript"

    - Closures & scope
    - Event loop & async/await
    - Prototypes & classes
    - ES6+ features
    - Memory management

=== "React"

    - Reconciliation & virtual DOM
    - useEffect pitfalls
    - Performance (memo, useMemo, useCallback)
    - State management patterns
    - Error boundaries

=== "CSS & layout"

    - Flexbox vs Grid
    - CSS specificity
    - BEM / CSS-in-JS
    - Responsive design patterns
    - Animation performance

=== "System design"

    - Component library architecture
    - Infinite scroll / virtualization
    - Real-time updates (polling vs WebSockets)
    - Caching strategies
    - Micro-frontend tradeoffs
